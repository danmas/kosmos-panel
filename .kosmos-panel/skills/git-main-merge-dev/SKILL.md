## What I do

Я выполняю следующие шаги:

1. Проверяю текущую ветку и статус репозитория
git status
2. Если есть незакоммиченные изменения в dev, запрашиваю сообщение для коммита - <MESSAGE>
3. Делаю коммит с предоставленным сообщением (если были изменения)
git commit -am "<MESSAGE>"
4. Переключаюсь на ветку main
git checkpout main
5. Проверяю нет ли текущей ветке(main) что-то незакомиченное
git status
6. Если есть - Выдаю сообщение и заканчиваю работу.
7. Выполняю merge ветки dev в main
git merge dev
8. Пушу изменения в удаленный репозиторий
git push

9. Возвращаюсь обратно в ветку dev
git checkout dev

Я всегда показываю вывод команд и спрашиваю подтверждение перед commit'ом (если это требуется по настройкам безопасности).

## When to use me

Используй меня, когда нужно обновить ветку main последними изменениями из dev и сразу запушить их в удаленный репозиторий. Это особенно полезно для быстрого деплоя или синхронизации основной ветки с разработкой.
