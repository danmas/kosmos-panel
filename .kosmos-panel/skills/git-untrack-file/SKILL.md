name: git-untrack-file
description: Удаление файла из-под контроля версий Git без удаления с диска
---

## Что я делаю

- Добавляю указанный файл в `.gitignore`, чтобы он не отслеживался в будущем.
- Удаляю файл из индекса Git с помощью команды `git rm --cached <file_name>`.
- Фиксирую изменения (commit) в текущей ветке.
- Помогаю применить эти изменения в других ветках (например, `main` и `dev`), чтобы файл был исключен из репозитория повсеместно.

## Когда использовать

Используйте этот навык, когда файл (например, `config.json`, `.env` или локальные логи) был по ошибке добавлен в репозиторий и должен быть удален из контроля версий, но сохранен локально на компьютере.

## Инструкции

1. **Добавление в .gitignore**: Откройте `.gitignore` и добавьте туда имя файла.
2. **Удаление из индекса**: Выполните команду `git rm --cached <путь_к_файлу>`.
3. **Коммит**: Создайте коммит с описанием "Remove <file_name> from tracking".
4. **Другие ветки**: Переключитесь на другие важные ветки (`git checkout <branch>`) и повторите шаги или объедините изменения.
