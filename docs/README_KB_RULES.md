# Правила ведения базы знаний (KB)

Краткий справочник правил. Полная инструкция с шаблонами: [ИНСТРУКЦИЯ_ВЕДЕНИЕ_БЗ.md](./ИНСТРУКЦИЯ_ВЕДЕНИЕ_БЗ.md).

---

1. **Единая точка входа**
   - Файл `KB/README_INDEX.md` — единственное оглавление БЗ.
   - Каждый `README_*.md` в `KB/` должен быть перечислен в одной из секций индекса.
   - В таблицах — все четыре колонки (Файл, Описание, Ключевые темы, Актуализация).

2. **Именование файлов**
   - Документация: `README_<ТЕМА>.md`. Исторические исключения допускаются.

3. **Актуализация**
   - Формат даты в колонке: `YYYY-MM-DD`.
   - Обновлять при содержательных изменениях (новые разделы, изменённая логика), не при правке опечаток.

4. **Структура документов**
   - В начале README — заголовок и краткое назначение (1–2 предложения).
   - Содержание соответствует «Ключевым темам» в индексе.
   - В конце — блок «Связанная документация» со ссылками на другие README.

5. **Ссылки**
   - Внутри KB — относительные: `[README_API.md](./README_API.md)`.
   - На код/тесты — от корня репозитория: `server/ws.js`, `tests/test_usa_v2.js`.
   - При переименовании/удалении README — обновить индекс и все ссылки.

6. **Синхронизация с кодом**
   - При добавлении значимой сущности (API, модуль, сценарий) — создать или обновить README и запись в `README_INDEX.md` (в т.ч. колонку Актуализация).

7. **Секции индекса**
   - Новый документ — в подходящую секцию; при необходимости временно в «Утилиты и прочее».

8. **Быстрый поиск**
   - В блоке «Быстрый поиск по темам» — только ключевые сценарии и 1–3 точки входа, без дублирования всех документов.

9. **Язык и стиль**
   - Краткие формулировки, таблицы для перечислений, единообразный стиль.

10. **Чеклист перед коммитом**
    - Изменения в KB → проверить обновление INDEX, при содержательных правках — дату Актуализации, работоспособность ссылок.
