# YouTube Download via yt-dlp (Windows + socks5 proxy)

## Когда активировать этот skill
- Пользователь просит скачать видео/аудио с YouTube
- Присылает ссылку вида https://www.youtube.com/watch?v=... или https://youtu.be/... или просто ID видео (11 символов)
- Упоминает слова: скачать, download, mp3, 360, максимум, качество, ютуб, youtube, yt

## Обязательные предусловия
- yt-dlp.exe должен лежать в PATH или в текущей директории
- Папка C:\ERV\youtube существует
- Прокси socks5://127.0.0.1:1086 работает (Tor / прокси-клиент запущен)

## Алгоритм (строго по шагам)

1. Перейти в целевую папку  
   ```
   cd /d C:\ERV\youtube
   ```  
   (используй /d чтобы сменить диск, если нужно)

2. Извлечь / нормализовать URL или ID  
   - Если в сообщении есть https://www.youtube.com/watch?v=XXXXXXXXXXX → взять как есть  
   - Если есть https://youtu.be/XXXXXXXXXXX → взять как есть  
   - Если есть чистый 11-символьный ID (например 8sHe_MruWcU, dQw4w9WgXcQ) → превратить в https://www.youtube.com/watch?v=ID  
   - Если несколько ссылок/ID — спросить, какую именно скачать (или предложить все по очереди)

3. Определить желаемый режим скачивания  
   Проанализировать сообщение пользователя и выбрать один из трёх вариантов:  

   | Ключевые слова / фразы в запросе               | Режим | Команда yt-dlp                                                                                   |
   |------------------------------------------------|-------|--------------------------------------------------------------------------------------------------|
   | максимум, max, лучшее, highest, full, 4k, 1080 | 1     | yt-dlp.exe --proxy socks5://@127.0.0.1:1086/ "<URL>"                                            |
   | 360, сжатое, маленькое, low, 360p              | 2     | yt-dlp.exe -S "res:360" --proxy socks5://@127.0.0.1:1086/ "<URL>"                               |
   | mp3, аудио, музыка, звук, audio, extract audio | 3     | yt-dlp.exe -x --audio-format mp3 --proxy socks5://@127.0.0.1:1086/ "<URL>"                      |
   | нет явного указания качества                   | 1     | (по умолчанию — максимальное качество)                                                           |

   Если пользователь написал что-то неоднозначное — спросить уточнение:  
   «Хочешь в максимальном качестве, в 360p или только mp3?»

4. Показать выбранную команду перед запуском  
   Вывести примерно так:  
   ```
   Собираюсь выполнить:
   cd /d C:\ERV\youtube
   <выбранная команда yt-dlp>

   Продолжить? (да / нет / изменить качество)
   ```

5. Если пользователь подтвердил — выполнить команду  
   Показывать прогресс / вывод yt-dlp в реальном времени.

6. После завершения  
   - Сообщить: «Скачивание завершено → файл(ы) в C:\ERV\youtube»  
   - Если имя файла известно — указать его  
   - Предложить: «Ещё видео?»

## Правила безопасности и ограничения
- Никогда не запускай без подтверждения пользователя (кроме очень коротких и очевидных запросов)
- Не скачивай автоматически плейлисты, если не сказано явно («весь плейлист», «все видео»)
- Если прокси не отвечает — сообщить: «Прокси 1086 недоступен — скачать не получится»
- Если yt-dlp.exe не найден — сообщить: «yt-dlp.exe не найден в PATH. Положи его рядом или добавь в PATH.»
- Не используй --update / --install автоматически
- Не добавляй --cookies / --username без явного запроса
