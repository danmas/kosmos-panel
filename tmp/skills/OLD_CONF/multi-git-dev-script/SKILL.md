# Multi-Repo Safe Commit & Push (только ветка dev)

## Когда использовать
- Пользователь хочет быстро закоммитить и запушить изменения во всех указанных проектах
- Фразы: "commit all", "закоммитить все репозитории", "push dev в проектах", "сделай коммит в kosmos"

## Фиксированный список папок (прошит в скрипте)
1. C:\ERV\projects-ex\kosmos-model  
2. C:\ERV\projects-ex\kosmos-vector  
3. C:\ERV\projects-ex\kosmos-vector-UI  

(Чтобы изменить список — отредактируй массив `folders` в `scripts/multi-repo-commit.js`)

## Как работает (строго по порядку для каждой папки)

1. Переход в папку
2. Проверка текущей ветки  
   → Если не `dev` → сообщение об ошибке и переход к следующей папке
3. Проверка статуса репозитория  
   → Если изменений нет → сообщение "нет изменений" и переход дальше  
   → Если есть изменения → вывод статуса + запрос сообщения коммита
4. После ввода сообщения (или "skip") →  
   - git add .  
   - git commit -m "<сообщение>"  
   - git push origin dev
5. Вывод результатов всех команд
6. После обработки всех папок — итоговая сводка

## Требования к среде
- Windows
- Node.js установлен и доступен в PATH (скрипт использует только встроенные модули — никаких npm install)
- git установлен и настроен (origin существует, права на push в dev)
- Папки существуют и являются git-репозиториями

## Запуск
Выполни скрипт напрямую по полному пути (замени на реальный путь к skill'у):

node "C:\Users\roman\.config\.kosmos-panel\skills\multi-git-dev-script\scripts\multi-repo-commit.js"

textСкрипт сам обработает все папки последовательно и будет взаимодействовать с пользователем через консоль (запросы сообщений).

## Правила безопасности
- Никогда не использует --force / --force-with-lease
- Запрашивает сообщение коммита отдельно для каждой папки с изменениями
- Пропускает папки при ошибках ветки, отсутствии изменений или по команде "skip"
- При любой ошибке (push rejected и т.д.) — выводит сообщение и продолжает со следующей папкой
- Не трогает другие ветки и не делает rebase/reset

Готов к использованию. Напиши "commit all projects" или "закоммитить все" — и начнём.
