# AI Промпты в Kosmos Panel

Как устроены и где используются системные промпты для ИИ.

---

## Автоматическое определение ОС

При подключении к серверу по SSH система автоматически определяет тип операционной системы командой `uname -s`. Результат (`linux`, `darwin`, `windows`) кэшируется на время сессии и добавляется в начало всех AI-промптов:

```
--- Target OS: Linux ---
Use POSIX/GNU commands. Examples:
- List files: ls -la
- Read file: cat file
- Find text: grep "pattern" file
```

Это позволяет AI генерировать команды, соответствующие целевой ОС:
- **Linux**: `ls`, `cat`, `grep`, `find`, `ps`, `systemctl`
- **macOS**: Аналогично Linux (POSIX/BSD команды)
- **Windows**: `dir`, `type`, `findstr`, `tasklist`, PowerShell cmdlets

## 1. Терминальные команды (AI_SYSTEM_PROMPT)

**Ключ:** `AI_SYSTEM_PROMPT` в `prompts.json` (fallback: `process.env` / config.json, иначе встроенный дефолт в `server/prompts.js`).

**Где срабатывает:** Пользователь вводит в WebSocket-терминале строку с префиксом `ai:` (например `ai: покажи файлы`). Обработка — `server/ws.js` (блок AI-запроса по WebSocket).

**Как используется:**
- В начало промпта автоматически добавляется информация о целевой ОС (см. выше).
- Системный промпт для чата: `getPrompt('AI_SYSTEM_PROMPT')`.
- Опционально добавляется контекст с **удалённой машины**: содержимое `./.kosmos-panel/kosmos-panel.md` или `~/.config/kosmos-panel/kosmos-panel.md` (читается по SSH с той же сессии).
- В `user` передаётся текст запроса пользователя (без префикса `ai:`).
- Ожидается **одна** shell-команда в ответ. Она выполняется на удалённом сервере по SSH, вывод возвращается в терминал.

**Итог:** один запрос к AI → одна команда (для правильной ОС) → один запуск на сервере.

## 2. Multi-step Skills (SKILL_SYSTEM_PROMPT_WITH_MEMORY)

**Ключи:**
- `SKILL_SYSTEM_PROMPT_WITH_MEMORY` — основной промпт для skills с поддержкой памяти и директив

Устаревшие (удалены):
- `SKILL_SYSTEM_PROMPT_WITH_ASK` — legacy промпт без памяти

Ключ в `prompts.json` (fallback: встроенный дефолт в `server/prompts.js`).

**Где срабатывает:** Запуск скилла из терминала или через API скиллов. Сборка промпта — `server/skill-ai.js` → `buildSkillSystemPrompt()`.

**Как используется:**
- В начало промпта автоматически добавляется информация о целевой ОС (см. выше).
- Базовый системный промпт: `getPrompt('SKILL_SYSTEM_PROMPT_WITH_MEMORY')`.
- К нему добавляются:
  - Опционально контекст с удалённой машины (как в п.1)
  - Блок **Active Skill** — содержимое SKILL.md выбранного скилла
  - Блок **Current Memory Index** — индекс долговременной памяти (если используется)
- Модель должна отвечать **строго** в одном из форматов: `[CMD]`, `[GET-FILE]`, `[ASK]`, `[ASK:optional]`, `[MESSAGE]`, `[DONE]`.
- Цикл: запрос к AI → разбор ответа → выполнение команды / чтение файла / показ вопроса / завершение. До появления `[DONE]` история дополняется и отправляется следующий запрос.
- **Фильтрация истории**: На основе `history_mode` из SKILL.md (full/last_n/system_only) перед отправкой в LLM фильтруется массив сообщений.

**Директивы skills:**
| Директива | Описание |
|-----------|----------|
| `[CMD] команда` | Выполнить shell-команду на сервере |
| `[GET-FILE] path` | Прочитать текстовый файл (whitelist расширений) |
| `[ASK] вопрос` | Запросить обязательный ввод пользователя |
| `[ASK:optional] вопрос` | Запросить опциональный ввод |
| `[MESSAGE] текст` | Показать сообщение и авто-продолжить |
| `[DONE] текст` | Завершить skill |

**Итог:** скилл — пошаговый сценарий с командами для правильной ОС, чтением файлов, вопросами к пользователю и настраиваемой историей LLM.

## 3. Справка по системе (AI_SYSTEM_PROMPT_HELP)

**Ключ:** `AI_SYSTEM_PROMPT_HELP` в config.json (или `process.env`). Не в prompts.json.

**Где срабатывает:** POST `/api/ai-help` в `server.js`. Вызывается из веб-интерфейса (вопрос пользователя по документации).

**Как используется:**
- Системный промпт: `process.env.AI_SYSTEM_PROMPT_HELP` (есть встроенный дефолт в коде).
- К нему дописывается блок «Документация системы»: содержимое файлов из `AI_HELP_CONTEXT_FILES` (config) или по умолчанию файлы из папки `KB/`.
- В `user` — текст вопроса. Ответ модели возвращается клиенту как текст справки (обычно на русском).

**Итог:** один запрос — один ответ по документации системы, без выполнения команд.

---

## 4. Общее

- **Автоматическое определение ОС:** При подключении по SSH выполняется `uname -s`. Результат кэшируется и добавляется во все AI-промпты.
- **Префикс команд в терминале:** по умолчанию `ai:`; настраивается через `AI_COMMAND_PREFIX` (config/env). Только такие строки обрабатываются как запрос к AI (п.1).
- **Контекстные файлы на удалённой машине** (`.kosmos-panel/kosmos-panel.md`, `~/.config/kosmos-panel/kosmos-panel.md`) используются для **терминального AI** (п.1) и **скиллов** (п.2).
- **Загрузка промптов:** `server/prompts.js` → `getPrompt(key)`. Источник для `AI_SYSTEM_PROMPT`, `SKILL_SYSTEM_PROMPT_WITH_ASK`, `SKILL_SYSTEM_PROMPT_WITH_MEMORY` — `prompts.json` в корне проекта; пример — `prompts.json.example`. Промпт справки (п.3) берётся только из config/env.

## 5. Файлы

| Файл | Описание |
|------|----------|
| `server/prompts.js` | Загрузка промптов, дефолтные значения |
| `server/ws.js` | Обработка AI-запросов в терминале, определение ОС |
| `server/skill-ai.js` | Сборка промптов для skills, `buildSkillSystemPrompt()` |
| `prompts.json` | Кастомные промпты (опционально) |
| `prompts.json.example` | Пример файла промптов |
